# 힌트 — 개념-먼저 빈칸 풀이 도우미

`/연습`으로 생성한 빈칸 골격을 풀다가 막혔을 때 호출한다.
**채점이 아니다.** 정답을 알려주는 게 목적이며, 그 전에 개념부터 쌓아준다.

---

## 트리거

- `/힌트` — 가장 최근 `practice/YYYY-MM-DD/` 폴더 자동 탐지
- `/힌트 JwtUtil` — 최근 폴더 안에서 해당 파일 (확장자 생략 OK)

---

## 동작 순서

### 1. 대상 파일 결정

1. `practice/` 폴더 스캔, 가장 최근 `YYYY-MM-DD/` 폴더 찾기 (날짜 내림차순)
2. 폴더 없거나 비어있으면 종료:
   > "먼저 `/연습` 으로 빈칸 골격 만들어줘"
3. 인자 처리:
   - 인자 없음 + 파일 1개 → 그 파일
   - 인자 없음 + 파일 여러 개 → 아래 형식으로 텍스트 목록 출력 후 채팅 응답 대기 (UI 선택지 금지):
     ```
     힌트 보고싶은 파일 번호를 입력해줘
     1. JwtUtil.java
     2. members-mapper.xml
     ```
   - 인자 있음 → 폴더 안에서 매칭 (대소문자 무시, 확장자 무시). 매칭 실패 시 폴더 안 실제 파일 목록 출력 후 종료

### 2. 원본 코드 찾기

`README.md` 제외한 연습 파일을 골랐다면, 동일 파일명의 **실제 코드**를 `src/`에서 찾는다.
- `Glob`으로 `src/**/{파일명}` 검색
- 실제 코드를 정답 기준으로 사용

### 3. 사용자가 채워놓은 빈칸은 무시

연습 파일에 사용자가 일부 채워놓은 상태일 수 있다.
**채운 거 보지 말고 원본(`src/`의 실제 코드) 기준으로만** 설명할 것.
잘못 채운 거 지적은 `/연습 채점`의 영역. 힌트에서 안 함.

### 4. 섹션 분할

파일을 논리 단위로 자동 분할. 예시:
- Java 클래스: 생성자 / 각 메서드 / 각 catch 블록
- XML mapper: 각 `<select>`/`<insert>` 쿼리
- Config 클래스: 각 `@Bean` 메서드

대략 5~7개 섹션이 적당. 너무 잘게 쪼개지 말 것.

### 5. 한 번에 다 출력 (섹션별로)

각 섹션마다 아래 6단계로:

1. **0번 섹션은 도메인 개념** — 파일 전체의 의미를 일상 비유로 (JWT면 "출입증" 비유 같은)
2. **섹션 제목** — `## N. 메서드명 — 한줄 요약`
3. **빈칸 코드 블록** — 그 섹션 부분만 발췌해서 `_____` 그대로 보여줌
4. **개념 설명** — "이게 왜 필요한지, 어떤 패턴인지" 메서드명 박기 전에 의도부터
5. **빈칸별 의도** — 각 `_____`이 뭘 의미하는지 자연어로 (메서드명 직접 박지 말고 "토큰의 주인을 설정하는 메서드" 식으로)
6. **정답 코드 블록** — 그 섹션의 완성본
7. (선택) **꼬리 질문 자답** — "왜 null이냐? → ..." 같이 자연스럽게 떠오를 의문 미리 풀어줌

### 6. 마무리 안내

마지막에:
> "이해됐으면 빈칸 채우고 `/연습 채점` 실행해줘."

---

## 출력 톤

- 한국어 음슴체 (`~함`, `~임`, `~됨`)
- 비유 적극 활용 (사용자가 비유로 이해 잘함)
- 빈칸 코드 → 설명 → 정답 코드 흐름 유지
- 메서드명을 설명 첫 줄에 박지 말 것. **의도 먼저, 메서드명은 정답 코드에서**

---

## 하지 말 것

- 사용자가 채운 거 보고 평가 (그건 `/연습 채점`의 영역)
- 정답만 코드 블록으로 뚝 던지기 (개념 설명 없이)
- 섹션 안 나누고 파일 전체를 한 덩어리로 설명
- UI 선택지 (`AskUserQuestion`) 사용 — 텍스트 목록 + 채팅 응답 방식만
- 분량 늘리려 잡담 추가 (각 섹션 핵심만)
