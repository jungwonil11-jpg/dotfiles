# 노션정리 — 프로젝트를 노션 학습 노트로 동기화

현재 프로젝트를 파악해 노션에 학습 노트를 생성·수정·보완한다.

## 노션 페이지 작성 스타일
- **음슴체** 사용 (예: "~임", "~함", "~됨")
- **비유** 적극 활용 — 낯선 개념은 익숙한 것에 빗대어 설명
- **표** 사용 — 비교, 속성 정리에 적합할 때
- **Callout 블록** 사용 — 핵심 요점, 주의사항, 팁 강조에 활용
- 코드 예제는 코드 블록으로 포함

## 각 하위 페이지 내부 구조
1. 개념 설명 (H2)
2. 핵심 규칙/문법 (H2 → H3 번호 목록)
3. 실습 코드 분석 (H2, 코드 블록, 핵심 예제 1~2개만)
4. 핵심 포인트 정리 (H2, 표)
5. 주의사항/팁 (Callout)

## 실행 순서

### 1. 프로젝트 파악
프로젝트의 파일 구조, 코드 파일, CLAUDE.md를 읽어 어떤 프로젝트인지 파악.

### 2. 노션 정리 방식 확인
CLAUDE.md에 `## 노션 정리 방식` 섹션이 있는지 확인.

**섹션이 없으면:**
- 프로젝트 구조를 보고 Claude가 직접 판단:
  - 노션 상위 페이지명 (예: "🐍 Python 강의")
  - 하위 페이지 구성 방식 (예: day별, 주제별, 파일별 등)
  - 정리 기준 (예: 코드 파일 기준, git 로그 기준 등)
- 판단한 내용을 CLAUDE.md에 자동으로 추가:
  ```markdown
  ## 노션 정리 방식
  - 노션 상위 페이지: (판단한 이름)
  - 하위 페이지 구성: (판단한 방식)
  - 정리 기준: (판단한 기준)
  ```
- 사용자에게 추가한 내용을 보여주고 맞는지 확인. 수정 요청 있으면 반영 후 진행.

**섹션이 있으면:**
- 해당 방식대로 진행.

### 3. 상위 페이지 찾기
노션에서 상위 페이지명으로 검색.
- 있으면 → 해당 페이지 사용
- 없으면 → 새로 생성

### 4. 하위 페이지 동기화
2단계에서 결정한 구성 방식대로 각 하위 페이지를 순서대로 처리:
- 노션에 해당 페이지 있는지 확인
  - 없으면 → 생성
  - 있으면 → 기존 내용과 비교 후 누락/변경 항목만 보완

### 5. 완료 보고
- 생성된 페이지 목록
- 수정된 페이지 목록
- 변경 없이 스킵한 페이지 목록
- 각 페이지 노션 링크 제공
